Video Game E-Commerce Ontology

This project contains the ontology for the e-commerce of video games. We used properties inspired from the Good Relation model and the schema.org model for the Offering object and a Video Game object. You can find the widoco-generated documentation for this project here.

Overview of the ontology

The classes and properties of our ontology was designed based on the following hierarchy that we came up with after extensive discussions:

Video games in our context will be represented as follows:

  • Video Game
    • Franchise
    • Genre
    • PEGI Rating
      • Rating
      • Criterion
    • Gameplay
      • Input Support
      • Game Mode
      • Achievements
  • Creative Work
    • Music By
    • Description
    • Title
    • Subtitle Languages
    • Dubbing Languages
    • Licence
    • Awards
    • Access Conditions
    • Contributors
    • Copyright
      • Copyright Holder
      • Copyright Notice
      • Copyright Year
    • Country of Origin
    • Status
    • Creation Date
    • Last Modification Date
    • Publication Date
    • Sponsors
    • Funders
    • isFree
  • Software
    • Version
    • Minimum System Requirements
      • CPU
      • OS
      • RAM
      • GPU
      • Storage Capacity
      • Networking Capabilities
    • Recommended System Requirements
    • Release Date
    • Developer
    • Publisher
    • Languages
    • EULA
    • Distribution Medium
    • Platform
    • Dependencies
    • Release Notes
    • Permissions
    • Software Add-ons

The above hierarchy is mainly inspired by the schema.org proposition

Offerings will have the following schema:

  • Offering
    • Serial Number
    • Accepted Payment Methods
      • Payment Method
      • Credit Card
      • hasMinValue
      • hasMaxValue
      • validThrough
    • Available at/or From
      • Location
    • Available Delivery Methods
      • hasValueFloat
      • Condition
    • Category
    • Delivery Lead Time
      • hasMinValue
      • hasMaxValue
      • hasUnitOfMeasurement
    • Description
    • Customer Requirements
    • Availability Duration
    • isAvailableInRegions
    • hasInventoryLevel
    • Price Specification
      • hasCurrency
      • hasValue
    • hasDeliveryCharge
    • hasUnitPriceSpecification
      • Price Type
      • Sale Status
    • hasWarrantyPromise
      • durationOfWarrantyInMonths
      • Warranty Scope
    • Includes
    • Name
    • Advertisement
      • Interviews
      • Articles
      • Events
      • Video Presentation
    • Customer Feedback
      • Rating
      • Reviews
      • Tags
    • Add-on
    • Valid From
    • Valid Through

The offering is mainly based on the Good Relations schema.

A more detailed presentation of the classes and properties can be found in the Widoco documentation, or you can simply load the video_game_ontology.ttl file in Protégé.

As a side-note, most of the elements we have added into these shemas come from the Steam storepage.

About browser issues on Widoco documentation

The result of executing Widoco is an html file. It as been tested it in Mozilla, IE and Chrome, and when the page is stored in a server all the browsers work correctly. If you view the file locally, we recommend you to use Mozilla Firefox (or Internet Explorer, if you must). Google Chrome will not show the contents correctly, as it doesn't allow XMLHttpRequest without HTTP. If you want to view the page locally with Google Chrome you have two possibilities:

a) Place the file in a server and access it via its URL (for example, put it in dropbox and access through its public url).

b) Execute Chrome with the following commands :

(WIN) chrome.exe --allow-file-access-from-files,

(OSX) open /Applications/Google\ Chrome.app/ --args --allow-file-access-from-files

(UNX) /usr/bin/google-chrome --allow-file-access-from-files
